## Section 5: Conviction Tier Decision Criteria

Use these thresholds for the conviction tier classification:

| Composite Score | Conviction Tier | Action |
|---|---|---|
| ≥ 0.70 | **Ready to Build** | Proceed to implementation planning |
| 0.40 – 0.69 | **Emerging** | Invest in validation; identify 2-3 assumptions to test |
| < 0.40 | **Not Ready** | Park; return after higher-conviction areas ship |

### Per-Dimension Minimums for "Ready to Build"

| Dimension | Minimum Score | Rationale |
|---|---|---|
| Vision Clarity | ≥ 0.60 | Can't build what you can't articulate |
| Market Understanding | ≥ 0.50 | Must know who the customer is |
| Technical Feasibility | ≥ 0.50 | Must have a credible path to build |
| Prioritization | ≥ 0.40 | Must know why this area first |
| Risk Awareness | ≥ 0.40 | Must have considered what could go wrong |

**Override rule:** If composite score ≥ 0.70 but any single dimension is below its minimum, downgrade to "Emerging" and flag the weak dimension.

## Section 6: DOCX Export Configuration

Pandoc is a required preflight dependency for DOCX generation:

```bash
pandoc --version 2>/dev/null | head -1
```

Export command:
```bash
pandoc {runtime_output_dir}/product-roadmap-{slug}.md \
  -o {runtime_output_dir}/product-roadmap-{slug}.docx \
  --from=markdown \
  --toc \
  --toc-depth=2 \
  --metadata title="Clockchain Product Discovery: {interviewee_name}" \
  --metadata author="Clockchain Product Team" \
  --metadata date="{ISO_date}"
```
